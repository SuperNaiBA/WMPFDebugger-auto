# WMPFDebugger 自动打开控制台功能说明

## 功能概述
本功能实现了检测到新开微信小程序后立即打开控制台的功能，无需手动点击"打开控制台"按钮。

## 技术实现

### 1. 多重检测机制
系统使用三种检测方式确保能够准确识别微信小程序的启动：

#### 进程检测（主要方式）
- 使用 `wmic process get commandline,processid` 命令
- 检测带有 `--wmpf` 参数的微信进程
- 记录进程历史，识别新启动的进程
- 检测频率：每秒一次

#### 系统日志检测（辅助方式）
- 使用 PowerShell 查询应用程序事件日志
- 检测微信相关的事件日志
- 检测频率：每秒一次

#### 后端日志检测（补充方式）
- 监听后端服务器的标准输出
- 匹配小程序启动关键词：
  - `微信小程序`
  - `WMPF`
  - `Mini Program`
  - `小程序启动`
  - `AppService started`
  - `小程序已加载`
  - `wxapkg loaded`

### 2. 响应速度优化
- 检测频率：从 3 秒改为 1 秒
- 打开控制台延迟：从 1 秒改为 0 秒（立即打开）
- 多进程支持：同时检测多个小程序进程

### 3. 智能功能
- **进程去重**：记录已检测的进程，避免重复打开控制台
- **自动刷新**：检测到小程序切换时自动刷新控制台
- **状态同步**：控制台关闭后可重新检测并打开

## 使用方法

### 自动模式（推荐）
1. 启动 WMPFDebugger 应用
2. 点击"启动调试器"按钮
3. 打开微信并启动任意小程序
4. 系统会自动检测并打开控制台

### 手动模式
如果自动检测失败，可以手动点击"打开控制台"按钮

## 故障排除

### 问题：自动打开控制台不工作
**可能原因：**
1. 微信未安装或未运行
2. 小程序启动方式不被支持
3. 权限不足（wmic 命令需要管理员权限）

**解决方案：**
1. 确保微信桌面版已安装并运行
2. 使用管理员权限运行应用
3. 检查运行日志查看具体错误信息

### 问题：检测速度慢
**解决方案：**
- 系统默认每秒检测一次，已是最优设置
- 确保系统资源充足，避免CPU占用过高
